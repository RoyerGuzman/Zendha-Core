Escribe la diferencia entre una API rest y una API soap
De las principales diferencias encontramos que los servicios SOAP requieren de la comunicación a través de XML para poder comunicarse de forma adecuada, lo cual requiere que la persona que implementa métodos para un API SOAP, requiera conocer el funcionamiento para poder hacer nuevas integraciones, en cambio bajo servicios REST, podemos tener mayor flexibilidad para implementación, siendo que tiene más flexibilidad para la comunicación entre sistemas, siendo JSON la forma de comunicación más usada, asimismo la implementación es más ágil debido a que esta no requiere conocer todo el aplicativo necesariamente para poder generar nuevas integraciones o módulos con diferentes funcionalidades

¿Qué es GIT?
Git es un manejador de versiones, nos permite tener un control en los cambios que generamos, así como también nos provee ramas, las cuales sirven para hacer algunas variaciones del proyecto base, principalmente se usan para generar una versión de código la cual tenga alguna modificación o nueva implementación del código y alguien pueda validar esos cambios antes de pasarlo a producción, esto disminuye el riesgo de algún cambio no considerado, así mismo nos permite que si algo no funciona correctamente después del último cambio, poder regresar a una vista anterior del código y devolver el aplicativo al funcionamiento anterior, y todo gestionado por GIT, así delegamos esa parte de saber que archivos fueron tocados en ese periodo para poder subirlos o modificarlos en el código del aplicativo sin tener el riesgo de olvidar aplicar algún ajuste. asi mismo otros de los grandes beneficios de GIT es que nos permite el trabajo colaborativo de forma ordenada y controlada.

Escribe la diferencia entre clase, objeto, array, método y función.
Para poder entender las diferencias entre los diferentes elementos, sería oportuno poder definirlos un poco más a detalle para poder comprender las diferencias, pues algunos son tipos de datos, otros nos aportan lógica y otro estructura…


  • Clase:
    Las clases en programación suelen utilizarse para tener una estructura base para la generación de objetos, los cuales son un tipo de dato, y con la introducción de los datos que esta necesite, nos devolverá un código con un formato específico y ordenado que se adaptara a la forma en la que el programa accede a la información de algún elemento

  • Objeto y Array:
    son tipos de datos no primitivos que nos aportan una colección de datos, pero la forma en la que estos lo guardan y cómo se accede a ellos es lo que los hace diferentes, en el caso de los Arrays, son una colección de datos que se van guardando segun la posicion en la que estos son almacenados, por lo que al querer acceder a su información lo haremos por un índice numérico, donde este apunta a la posición de la información que queremos recuperar, este tipo de dato no es el más adecuado para algunos casos de uso, en los cuales queremos acceder a alguna información a la cual no conocemos su posición, para ello se usan los Objetos, el cual en lugar de usar un índice numérico, utiliza un texto como referencia para acceder a la información, como puede ser un objeto “persona”, el cual tiene “nombre” y “edad”, utilizando alguna de las dos palabras referenciadas podremos acceder a la información de este fragmento sin necesidad de saber el orden de estos elementos dentro del objeto

  • Método y función:
    Aunque Funcionalmente un Método y una función pueden ser muy parecidos en el sentido que ambos pueden o no recibir datos y realizar alguna acción lógica al ser llamados, la función es aquel proceso lógico declarado fuera de alguna clase u objeto, mientras que el método estará ligado al objeto en cuestión, siendo este, una propiedad del mismo, llevando lógica enfocada a la alteración de este objeto, o de realizar alguna acción relacionada con información que esté ligada a este objeto, lo cual nos brinda flexibilidad de poder crear funcionalidades dinámicas pero con un control de estructura que nos brinda el mismo objeto, en cambio una función suele estar declarada en algún proceso, que puede tener diferente grado de encapsulamiento y no se restringe necesariamente a solo un tipo de comportamiento.

Escribe los comandos para agregar y commitear una carpeta por GIT

Tanto para agregar una carpeta/directorio como archivos independientes se usa el comando

git add "ruta y nombre de carpeta"

La diferencia es que al agregar la carpeta, nos agregara al stage todas las subcarpetas y archivos que estén dentro de ésta, en cambio al agregar un archivo se tiene que especificar el nombre y extensión del mismo, asi mismo tambien se puede usar el siguiente comando para agregar todos los archivos nuevos y modificados al stage

git add --all

posteriormente para que estos cambios se quede guardados se requiere hacer el commit del mismo, el cual sería:

git commit -m "Mensaje para el commit"
ejemplo: 
git commit -m "Corrigiendo bug de css"

Es importante tener en cuenta que antes de generar un commit tenemos que revisar la rama en la que estamos trabajando para que podamos gestionar correctamente los cambios y despliegues

Escribe la diferencia entre PHP CGI, fastCGI y FPM

Las diferencias entre las diferentes formas de poder utilizar php en nuestros servidores van desde temas de rendimiento hasta de complejidad de configuración, así que entremos un poco más a detalle

Las diferencias entre ellas residen desde el tiempo en que aparecieron y las problemáticas que fueron representando a lo largo del tiempo, siendo la siguiente un modelo mas completo para dar un mejor resultado en los procesos en el servidor, como son mejora de rendimiento y accesibilidad

CGI, siendo el modelo iniciar, traía características como fácil instalación, pero a costa de una afectación en el rendimiento, así como también se considera menos seguro por la forma en la que se ejecutan

FastCGI, fue la evolución del CGI, brindando como características principales el que los archivos php no fueran propiedad del mismo, así como una mejora en el rendimiento a costa de un mayor uso de memoria, pero incrementó la complejidad de configuración

FPM, siendo la última forma y la más recomendada, es una manera de manejar servidores que utilicen archivos php para trabajar, pero con mejor rendimiento y con mayor capacidad de tráfico

Escribe la diferencia entre paradigma POO y paradigma funcional
El paradigma funcional, como su nombre lo describe, está basado en bloques lógicos, donde enviamos información y este nos regresa un resultado a partir de ello, en cambio en el paradigma de Programación Orientado a Objetos, tenemos algunas herramientas adicionales, como son la Abstracción, el Encapsulamiento, la Herencia y el Polimorfismo, el cual nos brinda mayor grado de reutilizacion de codigo, e incremento de alguna funcionalidad de forma sencilla sin necesariamente afectar los objetos originales y sus propiedades, si no extendiendo las mismas a través de estas herramientas que nos brinda este paradigma
